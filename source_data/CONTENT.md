# Source Data

This folder contains the raw inputs for the multimodal EEG/fMRI fusion pipeline.
Paths are configured in `invoke.yaml`.

## Expected files (real data)

| File / Folder | Description |
|---|---|
| `phenotype.tsv` | One row per participant. Required columns: `participant_id`, `age`, `sex`, `site`, `diagnosis` (0/1). Additional columns are optional prediction targets. |
| `eeg_features.tsv` | *(EEG TSV mode)* Flat table: `participant_id` + one column per EEG feature. |
| `mne_output/` | *(EEG MNE mode)* MNE feature export — one subfolder per subject, each containing `{sub_id}_eeg_features.csv`. |
| `fmri_features.tsv` | *(fMRI TSV mode)* Flat table: `participant_id` + one column per unique connectivity value (upper triangle of the ROI x ROI matrix). |
| `halfpipe_output/` | *(fMRI Halfpipe mode)* Halfpipe output — one subfolder per subject, containing `{sub_id}_task-rest_run-{n}_desc-{strategy}_matrix.tsv` files. |

## Smoke data

`source_data/smoke/` is generated by `invoke generate-smoke-data` and contains all
four input formats above with 15 synthetic subjects and 20 ROIs. It is used by
`invoke run-smoke` and is deleted by `invoke clean-smoke`.

Note: TSV files in this folder are ignored by Git (see `.gitignore`).
